# Summary Methods Reference

## Choose summary style by purpose

A content item may have multiple usages. Combine methods when needed.

## 学习 / Learning

Focus:

- Concepts and definitions
- Mental models
- Core arguments
- Examples
- What to remember

Sections:

- 一句话结论
- 核心概念
- 关键机制
- 例子/证据
- 我应该记住什么
- 延伸问题

## 工具 / Tool

Focus:

- What it does
- Who it is for
- Key features
- Integration possibilities
- Pricing/limits if visible
- Risks/alternatives

Sections:

- 一句话判断
- 工具解决什么问题
- 关键能力
- 可用于我的哪些 workflow
- 接入成本/限制
- 是否值得试用

## 项目参考 / Project Reference

Focus:

- Reusable workflow/design/architecture
- Implementation details
- Dependencies
- Edge cases
- What can be copied or adapted

Sections:

- 可借鉴点
- 方案结构
- 实现步骤
- 风险和坑
- 可以马上做的下一步

## 灵感 / Inspiration

Focus:

- New angle
- Surprising insight
- Product/UX/content idea
- Connection to existing interests

Sections:

- 最有启发的一点
- 为什么有意思
- 可以衍生的想法
- 和我已有项目/兴趣的连接

## 研究 / Research

Focus:

- Claims
- Evidence quality
- Methodology
- Limitations
- Open questions

Sections:

- 研究问题
- 方法/数据
- 主要发现
- 证据强度
- 局限
- 后续问题

## 决策 / Decision Support

Focus:

- Options
- Pros/cons
- Constraints
- Recommendation
- Confidence level

Sections:

- 决策结论
- 选项对比
- 关键依据
- 风险
- 建议动作

## Universal required sections

Every summary must include:

```markdown
# Summary: <title>

## 一句话结论

## 简明摘要

## 关键要点

## 重要证据/细节

## 对之洋的启发

## 可执行动作 / 项目想法 / 后续问题

## 局限与不确定性

## 分类标签
```

## Tone

- Concise but not shallow.
- Prefer practical insight over generic summary.
- Explicitly separate original claims from inference.
- Include uncertainty when extraction is partial.
